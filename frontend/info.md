# Frontend Setup

## Запуск проекта

1.  **Предварительные требования**: Node.js (v18+ рекомендуется).
2.  **Установка зависимостей**:
    В корневой директории проекта (`frontend/`) выполните:
    ```bash
    npm install
    ```
3.  **Запуск сервера разработки**:
    ```bash
    npm run dev
    ```
    Frontend будет доступен по адресу: http://localhost:5173 (или другом, указанном в консоли).

## Конфигурация Backend

-   **API URL**: В `src/api.ts` настроен `baseURL: 'http://localhost:8000'`. Убедитесь, что на бэкенде настроен CORS, чтобы разрешить запросы с `http://localhost:5173`.

## Структура проекта

-   **Страницы**:
    -   `/register`: Регистрация пользователя (Student/Teacher). Данные отправляются как **Form Data** (`application/x-www-form-urlencoded`).
    -   `/login`: Вход в систему (получение JWT токена). Данные отправляются как **Form Data** (`application/x-www-form-urlencoded`).
    -   `/`: Главная страница (доступна только авторизованным пользователям). Отображает список классов (`/class`).
    -   `/class/:id`: Детальная страница класса.
-   **Стили**:
    -   Минималистичный дизайн, серый фон, зеленый акцент.
    -   Стили определены в `src/index.css`.
-   **Утилиты**:
    -   `src/api.ts`: Конфигурация Axios с интерцептором для добавления Bearer токена.

## Работа с токенами

-   Токен сохраняется в `localStorage` под ключом `access_token`.
-   При входе на главную страницу происходит проверка срока действия токена (expiration time). Если токен истек, пользователь перенаправляется на страницу входа.

## Устранение неполадок

-   **Ошибка CORS**: Если вы видите ошибки CORS в консоли, убедитесь, что на сервере (`localhost:8000`) разрешены запросы с `http://localhost:5173`.
-   **Ошибка 422**: Проверьте правильность отправляемых данных (валидация на бэкенде).
